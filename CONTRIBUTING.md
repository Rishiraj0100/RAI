# Contributing to RAI

Thank you for your interest in contributing to **RAI**! Your help is greatly appreciated. This guide explains how you can get involved, report issues, and submit changes.

---

## üìù How to Contribute

There are several ways to contribute:

1. **Report Bugs**
   - If you encounter a bug or issue, please open a new issue on GitHub.
   - Include as much detail as possible: steps to reproduce, screenshots, error messages.

2. **Suggest Features**
   - Have an idea to improve RAI? Open a feature request issue.
   - Describe what you want and why it would be helpful.

3. **Fix Bugs / Add Features**
   - Fork the repository and create a new branch for your work:
     ```bash
     git checkout -b my-feature-branch
     ```
   - Make your changes in Python, HTML, or Tailwind CSS as needed.
   - Test your changes locally to make sure everything works.
   - Commit your changes with a clear message:
     ```bash
     git commit -m "Add feature X / Fix bug Y"
     ```
   - Push your branch to your fork:
     ```bash
     git push origin my-feature-branch
     ```
   - Open a **Pull Request** on the main repository.

---

## üìö Coding Guidelines

- Write **clear, readable code**.
- Use consistent **naming conventions** for variables and functions.
- Keep changes **small and focused** for easier review.
- Comment your code where necessary to explain complex logic.
